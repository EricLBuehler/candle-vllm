系统概览

Candle-vLLM 是受 Python vLLM 启发、使用 Rust 重新实现的高性能 LLM 推理引擎。基于 Candle ML 框架，采用 Trait 驱动的模型管线架构，实现了从 HTTP 请求到 GPU 推理的完整流水线。

核心设计理念：

  • vLLM 风格的 PagedAttention 管理 KV 缓存
  • 连续批处理最大化 GPU 利用率
  • Trait 驱动的可扩展模型管线
  • CUDA 和 Metal 统一后端

架构图

HTTP API 层（Axum）— /v1/chat/completions, /v1/models, /v1/embeddings
↓
LLM Engine — 请求接收、批次编排、流式输出
↓
Scheduler — 连续批处理、分块预填充、抢占调度、前缀缓存
↓
DefaultPipeline — 模型前向传播、Logits 处理、Token 采样、工具解析
↓
硬件层 — CUDA / Metal / NCCL / FlashAttention / FlashInfer / CUDA Graph

请求处理流程

  1. 请求接收

    Axum HTTP handler 接收 OpenAI 格式的请求，解析消息、工具定义和采样参数，创建推理任务提交到 LLMEngine。

  2. 调度编排

    Scheduler 从等待/运行/换出队列中选择序列批次，分配/回收物理 KV 块。支持分块预填充（默认 8K）将长提示分批处理。

  3. 模型前向传播

    DefaultPipeline 准备批输入张量，调用模型的 forward() 方法。注意力层使用 PagedAttention 查表访问 KV 块，可选 CUDA Graph 加速。

  4. 采样与输出

    从 Logits 采样下一个 Token，通过工具解析器检测工具调用，通过 SSE 流式返回给客户端。

LLM Engine

LLMEngine 是系统的中枢，位于 src/openai/pipelines/llm_engine.rs。主要职责：

  • 接收异步推理任务
  • 调用 Scheduler 决定批次
  • 准备批输入张量并执行模型前向传播
  • 采样 Token 并流式返回结果

调度器

调度器实现了 vLLM 风格的连续批处理策略（src/scheduler/）：

  • 三级队列：等待队列、运行队列、换出队列
  • FCFS 抢占策略：单序列 → 重计算；多序列 → 换出到 CPU
  • 前缀缓存：复用已完成序列的 KV 块
  • 分块预填充：将长提示分为 8K 块交替处理

模型管线

DefaultPipeline（src/openai/pipelines/pipeline.rs）负责模型加载和前向传播。通过 HuggingFace config.json 的 architectures 字段（Safetensors）或 GGUF 元数据（general.architecture）自动匹配模型实现。

块引擎

BlockEngine（src/scheduler/block_engine.rs）管理逻辑 Token 块到物理 GPU/CPU 块的映射。支持引用计数和写时复制（COW），实现前缀缓存中的共享块。

项目结构

目录用途
src/main.rsCLI 入口、参数解析、HTTP 服务启动
src/lib.rs库根：导出 PagedAttention、设备/KV 缓存帮助函数
src/api.rs公共 Rust API：EngineBuilder、编程式推理
src/backend/底层后端：GGUF 加载、GPTQ/Marlin、CUDA Graph、自定义算子
src/openai/OpenAI 兼容层：HTTP 处理、请求/响应类型、采样、流式输出
src/openai/models/模型实现：各模型族的 Safetensors 和 GGUF 版本
src/openai/models/layers/共享层：Attention、MoE、MLA、DeltaNet、RoPE 等
src/scheduler/调度器：连续批处理、块分配、抢占、前缀缓存
src/mcp/MCP：stdio/HTTP 传输、客户端管理、工具发现
src/tools/工具解析：模型特定的流式解析器
examples/示例脚本和转换工具

添加新模型

  1. 创建模型文件

    在 src/openai/models/ 下创建新的 .rs 文件，实现模型结构体和 forward() 方法。使用共享层（attention.rs、mlp.rs 等）构建。

  2. 注册模型

    在 pipeline.rs 中添加 architectures 匹配分支，将 HuggingFace 架构字符串映射到你的模型实现。

  3. 添加对话模板

    在 src/openai/conversation/ 中添加模型族对应的对话模板/分隔符样式。

  4. 测试

    使用 cargo check 验证编译，然后用 HuggingFace 模型进行端到端测试。

Rust API 使用

Candle-vLLM 也可以作为 Rust 库使用，通过 EngineBuilder 进行编程式推理：

use candle_vllm::{EngineBuilder, ModelRepo};

let engine = EngineBuilder::new(ModelRepo::from_hf("Qwen/Qwen3-8B"))
    .with_devices(vec![0])
    .build()
    .await?;

let response = engine
    .chat(vec!["Hello, what is Rust?"])
    .await?;
