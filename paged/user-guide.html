支持的模型格式

Candle-vLLM 支持多种模型权重格式，通过 --m 参数自动检测：

格式描述用法
Safetensors (BF16/FP16)HuggingFace 标准格式--m repo_id
FP8 (Block-wise)SM90+ 块级量化--m repo_id-FP8
MXFP4 / NVFP44-bit 浮点量化--m repo_id-NVFP4
GGUF / GGML量化格式 (Q2K-Q8_0)--m file.gguf
GPTQ / AWQ / Marlin4-bit 权重量化--m gptq-repo
ISQ在线量化 (In-situ)--isq q4k

FP8/FP4 模型

# FP8 模型（需要 SM90+ 及 cutlass 特性）
candle-vllm --m Qwen/Qwen3.6-27B-FP8 --ui-server

# macOS/Metal 上的 FP8 模型
candle-vllm --m Qwen/Qwen3-4B-Instruct-2507-FP8 --ui-server

# NVFP4 模型
candle-vllm --m GadflyII/GLM-4.7-Flash-NVFP4 --ui-server

# MXFP4 模型
candle-vllm --m nm-testing/Qwen3-30B-A3B-MXFP4A16 --ui-server

GGUF 模型

GGUF 模型支持本地文件、本地目录和 HuggingFace 下载三种方式：

# 本地 GGUF 文件（推荐用 --m）
candle-vllm --m /home/data/model-Q4_K_M.gguf --ui-server

# 本地目录（自动检测 GGUF 文件）
candle-vllm --m /home/data/Qwen3.5-35B-A3B-GGUF/ --ui-server

# HuggingFace 下载（指定文件名）
candle-vllm --m unsloth/Qwen3-30B-A3B-Instruct-2507-GGUF \
  --f Qwen3-30B-A3B-Instruct-2507-Q4_K_M.gguf --ui-server

# HuggingFace 下载（子目录中所有 GGUF 文件）
candle-vllm --m unsloth/Qwen3.5-122B-A10B-GGUF \
  --f Q3_K_S --d 0,1 --ui-server
💡 多分片 GGUF

拆分的 GGUF 文件（如 model-00001-of-00005.gguf）会自动发现——本地和远程均支持。视觉模型的辅助文件（mmproj*.gguf）按需加载。

ISQ 在线量化

对未量化的模型进行即时量化，添加 --isq 参数即可：

candle-vllm --m Qwen/Qwen3.6-27B --isq q4k

可选量化级别：

级别精度
q4_0, q4_14-bit 基础量化
q5_0, q5_15-bit 量化
q8_08-bit 量化
q2k – q6kK-quant 量化族

GPTQ / AWQ / Marlin

# Marlin 兼容 GPTQ 模型（4-bit, 128-group, desc_act=False）
candle-vllm --m thesven/Llama-3-8B-GPTQ-4bit

# 将未压缩模型转换为 Marlin 格式
python3 examples/convert_marlin.py \
  --src /home/DeepSeek-R1-Distill-Qwen-14B/ \
  --dst /home/DeepSeek-R1-Distill-Qwen-14B-GPTQ_4bit-128g

# 将 AWQ 转换为 Marlin 格式
python3 examples/convert_awq_marlin.py \
  --src /home/Meta-Llama-3.1-8B-Instruct-AWQ-INT4/ \
  --dst /home/Meta-Llama-3.1-8B-Instruct-AWQ-INT4-Marlin/ \
  --bits 4 --method awq --group 128 --nk False

KV 缓存管理

Candle-vLLM 使用 PagedAttention 进行分块 KV 缓存管理。关键参数：

参数描述
--kvcache-dtypeKV 缓存类型：auto, fp8, turbo8, turbo4, turbo3
--kv-fractionGPU 剩余显存中用于 KV 缓存的比例（默认 0.6）
--mem固定 KV 缓存预算（MB）

TurboQuant KV 缓存压缩

基于 Walsh-Hadamard 变换的 KV 缓存压缩，在几乎不损失质量的前提下大幅延长上下文长度：

模式 描述 压缩比 推荐场景
turbo8 FP8 K + 4-bit V ~2.6× 质量与压缩的最佳平衡
turbo4 4-bit K + 4-bit V ~3.7× 质量与内存的均衡选择
turbo3 3-bit K + 4-bit V ~4.7× 最大内存节省 
# Turbo4 (4-bit KV 缓存, ~3.7x 压缩)
candle-vllm --w /data/Qwen3.5-27B-FP8/ --kvcache-dtype turbo4

# FP8 KV 缓存
candle-vllm --w /data/Qwen3.5-35B-A3B-FP8/ --kvcache-dtype fp8
⚠️ 注意

TurboQuant 使用原生 flash attention 内核（自动禁用 flashinfer）。支持 CUDA（SM70+）和 Metal（Apple Silicon）。MLA 模型（DeepSeek、GLM4）会自动回退到标准 KV 缓存。

前缀缓存

前缀缓存默认启用，将已完成请求的 KV 块复用于新请求的公共前缀，显著加速多轮对话和共享系统提示场景。

# 禁用前缀缓存
candle-vllm --m Qwen/Qwen3.6-27B-FP8 --disable-prefix-cache

# 限制前缀缓存大小
candle-vllm --m Qwen/Qwen3.6-27B-FP8 --prefix-cache-max-tokens 8192

多 GPU 推理

# QwQ-32B BF16 双卡
candle-vllm --d 0,1 --w /home/QwQ-32B/

# 多卡 GGUF
candle-vllm --d 0,1 --m unsloth/Qwen3.5-122B-A10B-GGUF --f Q3_K_S
💡 提示

GPU 数量（--d）必须为 2 的幂次（2、4、8）。

# 多线程模式（调试用途）
candle-vllm --multithread --d 0,1 --w /home/QwQ-32B/

# 如遇非法内存访问，禁用 P2P
export NCCL_P2P_DISABLE=1

多节点推理

通过 TCP 协调 NCCL 通信，无需 MPI，即可跨多台机器分布式推理：

# Master 节点 (192.168.1.100)
candle-vllm --d 0,1,2,3,4,5,6,7 --w /data/DeepSeek-R1-AWQ-Marlin/ \
  --num-nodes 2 --node-rank 0 \
  --master-addr 192.168.1.100 --master-port 29500

# Worker 节点 (192.168.1.101)
candle-vllm --d 0,1,2,3,4,5,6,7 --w /data/DeepSeek-R1-AWQ-Marlin/ \
  --num-nodes 2 --node-rank 1 \
  --master-addr 192.168.1.100 --master-port 29500

NUMA 绑定

sudo apt-get install numactl

# 8 GPU, 2 NUMA 节点
MAP_NUMA_NODE=0,0,0,0,1,1,1,1 \
  numactl --cpunodebind=0 --membind=0 \
  candle-vllm --d 0,1,2,3,4,5,6,7 --w /data/model/

MCP 与工具调用

Candle-vLLM 内置 Model Context Protocol (MCP) 支持，可以通过配置文件或命令行添加 MCP 工具服务器，实现 OpenAI 兼容的工具调用格式。

支持多种解析后端：qwen_coder、qwen、json、mistral

# 强制指定工具解析后端
candle-vllm --m Qwen/Qwen3.6-27B-FP8 --enforce-parser qwen

多模态模型

支持视觉语言模型，通过标准 OpenAI 图片格式发送图片 URL 或 Base64：

模型类型
Qwen3-VL视觉语言
Gemma 3 VL视觉语言
Mistral 3 VL视觉语言
LLaMA 4多模态

嵌入模型

通过 /v1/embeddings 端点提供文本嵌入服务，支持 mean 和 last pooling 两种模式。
